LLM 微调全景：LoRA、QLoRA、RLHF 与 DPO 的选择

Created2026-03-18|Updated2026-05-08
|Word Count:1.9k|Reading Time:7mins|Post Views:

「微调还是不微调？」这是每个 AI 应用开发者都会遇到的问题。

通用的大模型在很多场景下确实够用，但当你的业务需要模型具备特定领域的专业知识、遵循特定的输出格式、或者表现出特定的行为风格时，微调就变得必不可少。

但微调的世界很复杂——LoRA、QLoRA、全参数微调、RLHF、DPO……每种方法有不同的适用场景、计算成本和效果。这篇文章帮你理清思路。

微调不是什么魔法

先打破一个常见的迷思：微调不是给模型「灌输新知识」的最佳方式。

微调最适合做的事情是：

  1. 教模型特定的行为模式（如始终用 JSON 格式回复）
  2. 教模型特定的语言风格（如法律文书、医生问诊）
  3. 让模型遵循特定领域的推理方式（如代码审查、金融分析）

微调不太适合的事情是：

  • 让模型记住大量的事实知识（RAG 更适合这个）
  • 频繁更新知识（微调的频率通常以周或月为单位）

当你理解了这个边界，你就不会把微调用在错误的地方。

全参数微调 vs 参数高效微调

全参数微调

全参数微调就是在整个模型上训练——更新所有参数。

全参数微调只有在你有大量 GPU 和专业 ML 工程师时才应被考虑。绝大多数应用开发团队不应该碰它。

LoRA（Low-Rank Adaptation）

LoRA 的核心思想非常巧妙：不修改原始模型的权重，而是在旁边训练两个小矩阵 A 和 B。

关键的数字是 r（rank），通常取 8-64。这意味着 LoRA 只增加了模型参数量的 0.1%-1%，但效果接近全参数微调。

QLoRA：在单张消费级 GPU 上微调 70B 模型

QLoRA 在 LoRA 的基础上增加了两个技巧：

  1. 4-bit NormalFloat 量化：将原始模型权重量化为 4-bit，大幅减少内存
  2. 双重量化：对量化常数再做一次量化

这让你可以在单张 RTX 4090（24GB）上微调一个 65B 的模型——这在以前是不可想象的。

对齐训练：RLHF 与 DPO

当你不仅要改变模型的知识，还要改变模型的「行为」和「偏好」时，就需要对齐训练。

RLHF：让模型学会「什么是好的回答」

RLHF 是 ChatGPT 和 Claude 成功的核心原因之一。但它太复杂了——需要训练 Reward Model、运行 PPO、同时维护四个模型副本。

DPO：RLHF 的简化版

DPO（Direct Preference Optimization）跳过 Reward Model，直接从人类偏好数据中优化模型：

DPO 的优势是简单、稳定。你只需要准备「好回答 vs 差回答」的数据对，没有 Reward Model 和 PPO 的复杂性。强烈推荐从 DPO 开始做对齐训练。

微调的实战决策框架

微调数据的准备

微调最耗时的环节往往不是训练，而是准备数据。

数据质量 > 数据数量

对于 LoRA 微调，几百条高质量的示例通常比几万条低质量的示例效果更好。关键在于：

  • 每条示例准确反映你期望的输出
  • 覆盖主要的边界情况
  • 格式一致（不要有的用 JSON、有的用纯文本）

结语

微调的启动成本在持续降低——QLoRA 让你在消费级 GPU 上微调 70B 模型，DPO 让你不需要复杂的强化学习就能做对齐。这意味着微调正在从「大公司专属」变成「普通开发团队也能做的事情」。

但低门槛不等于低要求。微调成功的关键仍然是数据质量和评估标准——这两件事是任何工具都无法替代的。好的微调不是训练出来的，是准备出来的。
